# Robel auth 0.0.4-preview.30 upgrade and admin email lockdown

## Problem

Two related issues to solve:

1. The app is on `@robelest/convex-auth@0.0.4-preview.25`, which forced us to wrap GitHub through `arctic` + `OAuth(...)` and instantiate `Password` as a class. The 0.0.4-preview.30 release ships first-party lowercase factories (`github()`, `password()`, `google()`, `apple()`, `microsoft()`) with built-in profile fetch for GitHub. Staying on the older API leaves us carrying an unnecessary `arctic` dependency and custom GitHub profile code that the library now handles for us.
2. Admin gating today is a permissive table allowlist (`dashboardAdmins`) plus an optional strict email env var. The owner wants exactly one admin: whatever email is set in `DASHBOARD_PRIMARY_ADMIN_EMAIL`. Anyone else who completes GitHub OAuth currently bounces to `/?dashboardNotice=not-admin`. They should instead see the dashboard's demo mode and have their session signed out.

## Proposed solution

### 1. Upgrade auth

- Bump `@robelest/convex-auth` from `0.0.4-preview.25` to `^0.0.4-preview.30`.
- Remove `arctic` from `package.json`.
- Rewrite `convex/auth.ts` to use the lowercase factory pattern:
  - `password()` instead of `new Password()`.
  - `github({ clientId, clientSecret })` instead of `OAuth(new GitHub(...))` plus the manual profile fetcher (the library handles user + email lookup itself).
- Keep environment-driven conditional GitHub provider so deployments without `AUTH_GITHUB_ID`/`AUTH_GITHUB_SECRET` still boot.

### 2. Lock admin to a single email

- In `convex/dashboardAuth.ts`, change `isDashboardAdmin` so that when `DASHBOARD_PRIMARY_ADMIN_EMAIL` is set:
  - That email alone is admin.
  - The `dashboardAdmins` table is ignored (no need to also be in the table).
- When `DASHBOARD_PRIMARY_ADMIN_EMAIL` is unset, fall back to the existing `dashboardAdmins` table behavior so forks without env config still work.

### 3. Demo mode for non-admins

In `src/pages/Dashboard.tsx`, when an authenticated user is not the admin:

- Sign them out via the auth client (`authClient.signOut()`) per upstream guidance for app-level denied sessions. This stops the auth runtime from refreshing a session the app refuses to use.
- Render `<DashboardContent isDemo />` while the sign-out is in flight so they immediately see the read-only demo experience.
- Drop the `Navigate("/?dashboardNotice=not-admin")` redirect for the convex-auth path. Keep the WorkOS legacy redirect as is.

### 4. Skill update

Refresh `.cursor/skills/robel-auth/SKILL.md`:

- Mark 0.0.4-preview.30 as the current API surface.
- Note that lowercase factories (`github`, `google`, `apple`, `microsoft`, `password`, ...) are now first-party and ready to use.
- Drop the "no first-party github yet" warning.
- Keep the published-package reality-check section but update the version reference.
- Add the "denied session pattern" guidance (sign out + render denied view).

## Edge cases

- **Env var unset in legacy forks**: If `DASHBOARD_PRIMARY_ADMIN_EMAIL` is empty, the existing table-based allowlist must still work so we do not break other deployments. Handled by the early-return branch that only short-circuits when the env is set.
- **Email casing**: We already lowercase + trim. Keep that.
- **Admin email not yet on `userIdentity`**: When the JWT does not embed email, we already fall back to looking it up via `authComponent`. Keep that path.
- **Sign-out side effect during render**: Trigger sign-out in a `useEffect` so it does not run during the render phase. Track a local `isSigningOut` flag to avoid retriggering on re-render.
- **Race on first-load**: `isAuthenticated` and `isDashboardAdmin` queries may be `undefined` together. Continue to render `<LoadingState />` while either is loading.
- **Double-bootstrap**: With strict email mode, `bootstrapDashboardAdmin` becomes optional. Leave the mutation in place since it remains useful for forks without strict email.

## Verification steps

1. `npm install` succeeds without `arctic` and pulls preview.30.
2. `npx tsc --noEmit` passes (`tsconfig.json`) and `npx tsc -p convex/tsconfig.json --noEmit` passes.
3. `convex/auth.ts` no longer imports from `arctic`.
4. `npx convex dev` boots without push errors.
5. Sign in with the configured admin email → full dashboard renders.
6. Sign in with any other GitHub account → demo mode renders, session is cleared, refresh stays on demo mode.
7. Unset `DASHBOARD_PRIMARY_ADMIN_EMAIL` → fall back to `dashboardAdmins` table works as before.
